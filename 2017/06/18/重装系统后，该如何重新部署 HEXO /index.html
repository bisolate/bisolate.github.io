重装系统后，该如何重新部署 HEXO|GitHub

Posted by Benjamin Benjamin on Jun 18, 2017

更新到 Win 10 的 16215 版之后，系统开始各种抽风，显卡崩了似的不工作，分辨率拉到 1024*768 变成大胖脸，每天 N 次 ctf 报错，Shadowsock 经常莫名丢配置，所有蓝牙音箱也从此连不上了。无计可施，只有出终极大招——重装系统。不知为何，重装的三个选项中，能保留文件的两个都灰的不可用，只能彻底重来。也好，从零开始，问题也解决得再彻底。

但是重装也往往带来意想不到的新问题。比如我自制的输入法居然不可用了，最后重新生成一次才算正常。最吓人的是，我的这个博客也一度罢工了，一副要跟我断绝联系的架势。我之前是看别人的攻略才成功部署的这个博客，中间其实也重来过一次，我以为按着攻略重来一次，也就能照旧使用了，没想到这个没有成功。可能是时隔太久，有些关键步骤我给忘了。所以，这次我吸取教训，简单地把过程写一遍吧。

重装之前，千万要先备份博客文件夹！

重装之后，第一步是先把两个关键的环境装好：

  1. Node.js
  2. Git

第二，安装 Hexo：从开始菜单中找到“Git Bash”，先设置两个重要参数：

$ git config –global user.name “John Doe”
$ git config –global user.email johndoe@example.com

这俩是对应 GitHub 的账户，填错了或忘了设置，就会出现“Please tell me who you are.”之类的报错。如果记忆模糊，可以去看原博客文件夹下的“_config.yml”，看原来怎么写。登记过多个 email 的时候，尤其容易弄错。

第三，新建个文件夹，比如 blog，然后在空白处右键“Git Bash Here”，初始化博客：

hexo init

似乎直接在原文件夹下 init 会报错，所以新建一个，后面要 deploy 的时候把原文件夹下的文件一股脑 copy 过来就是了。

第四，重新生成 deploy key：

ssh-keygen -t rsa -C “Github的注册邮箱地址”

一路 enter，万一生成过一次，在问要不要 overwrite 的时候选“y”，最终得到这个：

Your public key has been saved in /c/Users/user/.ssh/id_rsa.pub.

找到这个 id_rsa.pub，用记事本打开它，复制全部内容。

第五，网页登陆 GitHub，找到*.github.io项目的“Settings/Deploy keys”，然后“Add deploy key”，Title随意，Key 的内容是刚才复制的内容，记得勾选“Allow write access”，不然后面会遇到权限问题。

最后，把原文件夹的文件全部复制到新文件夹，然后就可以正常发文了。

常用 Hexo 命令：

  • 清除缓存：hexo clean
  • 新建文章：hexo n “博文名”
  • 生成文件：hexo g
  • 部署博客：hexo d

通常“吃个蛋”就足够了：

hexo clean && hexo g && hexo d

遇到“error deployer not found:github”的时候，可以执行：

npm install hexo-deployer-git –save
